# Шаг 8 — Тест: прогресс «модуль пройден»

**Предыдущий:** [step-07-student-views.md](step-07-student-views.md) · **Следующий:** [step-09-quiz-test.md](step-09-quiz-test.md)

## Зачем

Кнопка «Отметить модуль пройденным» обновляет progress bar и галочки ✓ в sidebar. При 100% модулей срабатывает `on_course_completed()` (сертификат — шаг 10).

> Код уже реализован на шаге 7 (`ModuleCompleteView` + `mark_module_complete` в services). Этот шаг — **только ручная проверка**.

## Подготовка

1. Войдите как **`student`**, записанный на курс **Python Basics** (шаг 7).
2. У курса должно быть **не менее 2 модулей** (лучше 3 — так проще видеть процент).
3. Сервер запущен: `python manage.py runserver`

## ✅ Ручная проверка (сделайте сейчас)

### Начальное состояние

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Открыть `/students/course/<id>/` | Progress bar показывает **0%** |
| ☐ | Sidebar — список модулей | Ни у одного модуля нет галочки ✓ |
| ☐ | Первый модуль выбран | Контент модуля отображается в основной области |

### Отметка одного модуля

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Нажать «Отметить модуль пройденным» | POST-запрос, страница перезагружается |
| ☐ | URL после редиректа | `/students/course/<id>/module/<module_id>/` |
| ☐ | Sidebar — текущий модуль | Появилась галочка ✓ |
| ☐ | Progress bar | Обновился (например **33%** при 3 модулях, **50%** при 2) |
| ☐ | Текст progress bar / подпись | Соответствует «1 из N модулей» |

### Повторное нажатие

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Снова нажать «Отметить модуль пройденным» на том же модуле | Страница OK, прогресс **не** уменьшился |
| ☐ | Progress bar | Тот же процент, что после первого нажатия |
| ☐ | Sidebar | Одна галочка ✓ на этом модуле (не дублируется) |

### Несколько модулей

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Перейти ко второму модулю (sidebar) | Контент второго модуля, progress bar тот же % |
| ☐ | «Отметить модуль пройденным» для второго | Progress bar вырос (например **66%** при 3 модулях) |
| ☐ | Sidebar | ✓ у двух модулей |
| ☐ | Переключиться между модулями в sidebar | ✓ сохраняются у пройденных |

### Проверка в БД

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Shell — количество ModuleProgress | Соответствует числу нажатых «пройден» |

```bash
python manage.py shell -c "
from django.contrib.auth.models import User
from students.models import ModuleProgress
print(ModuleProgress.objects.filter(user__username='student', completed=True).count())
"
```

### Граничные случаи

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Студент не записан на курс → POST на `/students/course/<id>/module/<mid>/complete/` | **404** |
| ☐ | Гость → `/students/course/<id>/` | Редирект на login |
| ☐ | Обновить страницу (F5) после отметки | Progress и ✓ сохранились |
