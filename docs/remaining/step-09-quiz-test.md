# Шаг 9 — Тест: прохождение викторин

**Предыдущий:** [step-08-progress-test.md](step-08-progress-test.md) · **Следующий:** [step-10-certificates.md](step-10-certificates.md)

## Зачем

Студент отвечает на вопросы викторины, видит score и статус passed/failed.

> Код уже реализован на шаге 7 (`QuizTakeView`, `QuizResultView`). Этот шаг — **только ручная проверка**.

## Подготовка

1. Войдите как **`student`**, записанный на **Python Basics**.
2. В одном из модулей должна быть **викторина** с **не менее 2 вопросами**, у каждого — несколько ответов, один помечен `is_correct` (создано на шаге 4).
3. Запомните `pass_percent` викторины (по умолчанию **70**).
4. Сервер: `python manage.py runserver`

## ✅ Ручная проверка (сделайте сейчас)

### Открытие теста

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | `/students/course/<id>/` → модуль с викториной | В контенте блок «Викторина» / ссылка «Пройти тест» |
| ☐ | «Пройти тест» | URL `/students/course/<id>/quiz/<quiz_id>/` |
| ☐ | Страница теста | Заголовок викторины, список вопросов |
| ☐ | Каждый вопрос | Radio-кнопки с вариантами ответов |
| ☐ | Кнопка «Отправить» / Submit | Форма готова к отправке |

### Успешное прохождение (все ответы верные)

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Выбрать **правильный** ответ для каждого вопроса | Radio отмечены |
| ☐ | Отправить форму | Редирект на `/students/course/<id>/quiz/<quiz_id>/result/` |
| ☐ | Страница результата | **«Тест сдан!»** (или аналог в шаблоне) |
| ☐ | Score | **100%** (или 100) |
| ☐ | Статус passed | Отображается как успех |

### Неуспешное прохождение

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | «Пройти тест» снова (или вернуться по ссылке) | Форма с вопросами |
| ☐ | Выбрать **неверные** ответы (или частично верные) | — |
| ☐ | Отправить | Редирект на result |
| ☐ | Score | **< pass_percent** (например 0% или 50%) |
| ☐ | Статус | **«Тест не сдан»** (или аналог) |

### Повторная попытка и обновление результата

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Снова пройти тест с **все верными** ответами | Result: passed, score обновился |
| ☐ | Shell — QuizResult | Одна запись на user+quiz (не дубликаты) |

```bash
python manage.py shell -c "
from django.contrib.auth.models import User
from courses.models import QuizResult
u = User.objects.get(username='student')
print(QuizResult.objects.filter(user=u).values('quiz_id', 'score', 'passed'))
"
```

### Права доступа

| ☐ | Действие | Ожидаемый результат |
|---|----------|---------------------|
| ☐ | Гость → `/students/course/<id>/quiz/<quiz_id>/` | Редирект на login |
| ☐ | Студент **не** записан на курс → тот же URL | **404** |
| ☐ | Result без прохождения теста → `/students/course/<id>/quiz/<quiz_id>/result/` | **404** (нет QuizResult) |
